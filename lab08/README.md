# Лабораторная работа 8.
## Пуассоновский поток. События на сервере
Моделирование потока заявок на сервер.

**Задание:**
- событие — поступление заявки;
- определить число заявок за интервал времени T;
- построить распределение числа заявок;
- вычислить среднее и дисперсию;
- сделать вывод.

### Реализация

Для моделирования использовалась модель простейшего пуассоновского потока.

Простейший поток обладает следующими свойствами:
- Независимость: события наступают независимо друг от друга;
- Ординарность: в один момент может наступить только одно событие;
- Стационарность: число событий, которые наступят в некотором интервале времени, не зависит от начальной точки интервала (а только от его длины).

при моделировании случайного потока задаётся начальный момент времени, после чего моделируется интервал до следующего события, затем до следующего и так далее. Для простейшего потока интервалы между событиями имеют экспоненциальное распределение с параметром $\lambda$. 

Параметр $\lambda$ задаёт интенсивность входящего потока, то есть среднее количество заявок, поступающих в единицу времени.

Интервал времени до следующей заявки вычисляется по формуле:

$$
\tau = -\frac{\ln(\alpha)}{\lambda}
$$

где:

$\alpha \in (0, 1)$ — случайное число;
$\lambda$ — интенсивность потока;
$\tau$ — время до поступления следующей заявки.

---
Генерация интервала между соседними заявками реализована в методе `generate_interval`:
```python
def generate_interval(self):
    return -math.log(random.random()) / self.lambda_  # Формула
```
---
Моделирование потока заявок на интервале времени [0, T] реализовано в методе `simulate`:
```python
def simulate(self, T):
    t = 0
    count = 0

    while t < T:
        t += self.generate_interval()

        if t <= T:
            count += 1

    return count    
```
Сначала текущее время `t` равно нулю, затем последовательно генерируются интервалы времени между заявками. Каждый интервал добавляется к текущему времени, и если момент поступления не превышает `T`, то заявка учитывается. Если момент времени стал больше `T`, то моделирование интервала завершается.

---
Метод `experiment`выполняет моделирование потока несколько раз. Каждый раз поток моделируется на интервале длины $T$, после чего сохраняется число заявок, поступивших за этот интервал.
```python
def experiment(self, T, total_time):
    results = []
    ex_count = int(total_time / T) # N = total_time / T

    for _ in range(ex_count):
        count = self.simulate(T)
        results.append(count)

    return results
```
Пользователь задает $\lambda$, длину интервала наблюдения T и общее время моделирования total_time.
Количество интервалов наблюдения вычисляется как: 

$$ 
N = \frac{\text{total\_time}}{T} 
$$

---

После завершения моделирования получается выборка $x_1, x_2, \dots, x_N$, где $x_i$ — количество заявок, поступивших интервал времени $T$.

Далее мы вычисляем среднее число заявок и дисперсию выборки.

---
Эмпирическое распределение показывает, с какой относительной частотой встречалось каждое количество заявок.

Эмпирическая вероятность определяется как:

$$ \hat p_k = \frac{n_k}{N} $$

где:
- $k$ — число заявок за интервал T;
- $n_k$ — количество интервалов, в которых было ровно k заявок;
- $N$ — общее количество интервалов наблюдения.

Построение эмпирического распределения реализовано в методе get_distribution():
```python
def get_distribution(self, results):
    distribution = {}

    for x in results:
        if x not in distribution:
            distribution[x] = 0
        distribution[x] += 1

    for x in distribution:
        distribution[x] /= len(results)

    return dict(sorted(distribution.items()))
```

В ходе лабораторной работы был реализован простейший пуассоновский потом заявок на сервер.

Событием в модели является поступление заявки. Поток моделируется во времени: сначала генерируется интервал до следующей заявки, затем этот интервал прибавляется к текущему времени.
